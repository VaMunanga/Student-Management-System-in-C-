#FULL TECHNICAL REPORT OF PROJECT

This report outlines how each programming concept was applied to build a robust, data-driven Windows Forms application for managing student records.
1. Windows Forms Application & UI Design (Tasks 1, 2, & 8) I developed the application using Windows Forms to create a simple desktop interface. To design the UI, I used TextBoxes to accept text input (like the student's name), and a ComboBox so the user can easily select a course without making spelling errors. I also implemented advanced controls: a DateTimePicker to guarantee the enrollment date is formatted perfectly, and a ListBox to act as a live system log so the user can see everything the program is doing in real-time. The DataGridView acts as the main visual hub for the data.
2. Database Connectivity & CRUD Operations (Tasks 4, 5, & 6) To ensure data persistence, I created a local SQL Server database (HIT_StudentDB). I connected the C# application to this database using the SqlConnection class and a secure connection string. For the CRUD operations, I used SqlCommand with parameters (like @Name and @Date). Using parameters is crucial because it prevents SQL Injection attacks. I implemented the Insert, Update, and Delete functions using ExecuteNonQuery(). To view the records, I used a SqlDataAdapter to fill a DataTable, which automatically populates the DataGridView. I also added a CellClick event to the grid, allowing users to navigate simply by clicking a row, which automatically fills the textboxes.
3. Error Handling & Custom Exceptions (Tasks 3 & 9) Software should not crash when a user makes a mistake. To handle errors, I wrapped my database operations in try-catch blocks. However, I went further by creating a custom exception class named InvalidStudentDataException. If a user attempts to insert a record with a blank name, the program throws this specific exception. The system catches it and displays a polite warning message, preventing bad data from entering the database and preventing the application from crashing. I also added safety nets, like a confirmation warning box before permanently deleting a record.
4. File Handling: Export and Import (Task 7) To allow data portability, I implemented a file handling system using StreamWriter and StreamReader. For exporting, the program loops through the live DataGridView, flattens the relevant data (Name, Course, Date) into a comma-separated string, and writes it to a .txt file using StreamWriter. For importing, I used File.ReadAllLines to pull the text file back into the application, split the strings at the commas, and dynamically insert those records back into the SQL database. I used SaveFileDialog and OpenFileDialog so the user can control exactly where the file goes.
5. Packaging and Deployment (Task 10) Once the system was fully functional I used Visual Studio's Publish feature to package the application. This compiled the code into an executable installer (setup.exe) allowing the program to be deployed and installed on other Windows machines easily.
